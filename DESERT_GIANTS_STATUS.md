## Что сделано сегодня

### 1. Инфраструктура
- Создана изолированная папка `scenes/desert-giants/` (не попадает на сайт из `master`).
- Настроен локальный HTTP-сервер (`start-server.bat` / `.ps1`) для разработки.
- Добавлен Playwright для автоматических скриншотов (`node_modules` закоммичены в ветке).
- **Git:** ветка `feature/desert-giants` запушена на GitHub. В `master` добавлен `.gitignore`, исключающий `scenes/` и `__UE_web/` из деплоя.

### 2. Загрузка ассетов
- Скопированы HDR (`desert_sunset_2k.hdr`), GLB (`desert_scene.glb`) и billboard-спрайты в `assets/`.
- Добавлен `LoadingManager` с прогресс-баром и fallback-скрытием через 15 сек.
- Проверка `file://` протокола с предупреждением пользователю.

### 3. Сцена Three.js
- **Рендерер:** `ACESFilmicToneMapping`, `PCFSoftShadowMap`, `alpha: true`.
- **Камера:** `PerspectiveCamera(45)`, старт `(15, 8, 20)`, `OrbitControls` с damping, clamp polar angle.
- **Свет:** DirectionalLight (солнце + fill) + AmbientLight, тени `1024×1024` (desktop).
- **Окружение:** HDR `EquirectangularReflectionMapping` для `scene.environment`.
- **GLB:** загружен `desert_scene.glb`, включены `castShadow` / `receiveShadow`.

### 4. Визуальные эффекты
- **Пыль/туман:** 5 плоскостей с `ShaderMaterial` (simplex noise) + 1200 точек `Points` с аддитивным блендингом.
- **Перекраска по bounding box:** логика определения пустыни / гигантов / мелких объектов по `size.y`.
- **Billboard-спрайты:** два `THREE.Sprite` для дальних гигантов (возможно, устарели после обновления GLB).

## Главная проблема: белые фигурки-артефакты

### Что это
В сцене рядом с основными гигантами (внутри `mesh_0`) присутствуют небольшие белые фигуры, которые пользователь отметил к удалению. Они похожи на пастухов / людей.

### Результаты расследования

| Меш | Размер | Позиция (центр) | `map` | Примечание |
|-----|--------|-----------------|-------|------------|
| `mesh_0` | 58×17×92 | (0, 8.6, 0) | ✅ | **Основные гиганты + белые фигурки внутри** |
| `mesh_0_1` | 198×0.4×216 | (0, 1.6, 0) | ✅ | Пустыня |
| `mesh_0_2` | 20×6.7×10 | (-51.8, 5.1, 0.5) | ❌ | Белый дальний объект (виден в центре кадра) |
| `mesh_0_3` | 20×0.8×2.7 | (-51.8, 8.9, -4.3) | ❌ | Белый дальний объект |
| `mesh_0_4` | 15×1.7×2.8 | (-51.8, 2.3, 3.9) | ✅ | Белый дальний объект |

#### Критические находки
1. **`mesh_0` — единый `Mesh` с `BufferGeometry`, без дочерних объектов.** `geometry.groups = []`, `material` — не массив.
2. **Все материалы имеют `onBeforeCompile = true`** (кастомный шейдер, вероятно, из Houdini). Это блокирует стандартное поведение:
   - `material.color.set()` — игнорируется.
   - `material.wireframe = true` — применяется только к «основной» части геометрии, белые фигурки остаются сплошными.
   - `map = null` + `color.set(0x000000)` — гиганты тёмнеют, фигурки остаются белыми.
3. **`geometry.attributes` содержит Houdini-атрибуты:** `_coverage`, `_mask`, `_curvature`, `_numnonpacked`. Все они равномерны (`_mask` = 0 везде), поэтому разделение цвета происходит **не через vertex attributes**, а на уровне шейдерного кода в `onBeforeCompile`.
4. **`setDrawRange(0, 0)` скрывает весь `mesh_0` целиком** — это доказывает, что белые фигурки действительно находятся внутри той же геометрии.
5. **`mesh_0_2/3/4`** — отдельные белые mesh'и без текстур (`map: false`). Их можно удалить кодом (они не нужны в сцене).

### Почему стандартные методы Three.js не работают
`GLTFLoader` применил `onBeforeCompile` к `MeshStandardMaterial`, полностью переопределив шейдерный код. Стандартные uniform'ы (`diffuse`, `map`, `color`) либо не используются, либо используются избирательно. Поэтому любые попытки «покрасить» или «скрыть» через свойства материала бессильны.

## Варианты решения (на завтра)

### Вариант A — рекомендуемый: пересобрать GLB в Houdini / Blender
- **Что сделать:** открыть исходный `scene.glb` (из `__UE_web/`) в Blender или Houdini, отделить белые фигурки в отдельный named mesh (например, `shepherds`), экспортировать обновлённый GLB.
- **Плюсы:** 100% контроль, чистый код, никаких хаков.
- **Минусы:** требуется ручная работа в 3D-редакторе.

### Вариант B: программное удаление вершин по bounding box
- **Что сделать:** на стороне JS прочитать `geometry.attributes.position`, найти вершины в области белых фигурок (примерно `y < 2`, `|x| < 5`, `|z| < 5`), пересобрать `BufferGeometry` без них.
- **Плюсы:** без Blender/Houdini.
- **Минусы:** сложно, хрупко, может сломать ноги гигантов.

### Вариант C: `renderer.localClippingEnabled` + `clippingPlanes`
- **Что сделать:** добавить `THREE.Plane`, отсекающий область между ногами гигантов.
- **Плюсы:** просто в коде.
- **Минусы:** может отрезать части ног, выглядит как хак.

### Вариант D: модифицировать `onBeforeCompile`
- **Что сделать:** вставить `discard` в шейдер на основе world-координат (через `clipping` внутри шейдера).
- **Плюсы:** элегантно.
- **Минусы:** нужно реверс-инжинирить шейдер Houdini; сложно.

## Примечания

- **HTTP-сервер обязателен:** сцена не работает с `file://` из-за CORS на загрузку GLB/HDR.
- **Фоновые таски:** иногда падают по таймауту (120 с / 3600 с) — это известная проблема окружения, не критично.
- **Билборды:** после обновления GLB (в котором нет плоскостей) спрайты `giant_billboard_01/02` скорее всего избыточны — можно удалить.
