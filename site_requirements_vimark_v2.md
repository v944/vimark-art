# Требования к сайту-портфолио художника-иллюстратора
## Адаптированная версия под vimark.art (v2.0)

**Назначение:** Этот документ — доработанный чек-лист для аудита сайта иллюстратора Max Mitenkov (vimark.art). Все пункты адаптированы под международное арт-портфолио с упором на SEO, UX и конверсию. При анализе сайта проходи по каждому пункту и помечай `[x]` для выполненных, `[ ]` для нарушенных.

---

## Блок 1. Техническое SEO и индексация

- [ ] **Семантическая вёрстка** – один `<h1>` на страницу, иерархия `<h2>`–`<h3>`, теги `<article>`, `<section>`, `<main>`, `<header>`, `<footer>`.
- [ ] **ЧПУ-адреса** – URL вида `vimark.art/work/elf-sketch` или `vimark.art/en/portfolio/elf-sketch` для мультиязычности. Не допускаются параметры `?p=125`.
- [ ] **Mobile First / Responsive** – адаптивная вёрстка с брейкпоинтами (320, 768, 1024, 1440+ px), контрольные точки для арт-галерей.
- [ ] **Скорость загрузки (Core Web Vitals)** – LCP < 2.5 с, CLS < 0.1, FID/INP < 200 мс. Lazy loading для всех изображений ниже первого экрана.
- [ ] **robots.txt** – запрещает `/admin`, `/.env`, `/cgi-bin`, `/wp-config` (если есть). Явно разрешён доступ к `/images`, `/uploads` для всех `User-agent: *`.
- [ ] **XML Sitemap** – автогенерация с приоритетами: проекты > 0.9, категории > 0.7, About/Contact > 0.5. Отдельный sitemap для изображений (`image-sitemap.xml`).
- [ ] **Микроразметка Schema.org** – обязательны:
  - `VisualArtwork` или `ImageObject` для каждой работы (автор, название, дата, техника, лицензия);
  - `Person` для страницы About (имя, должность, url, sameAs — ссылки на соцсети);
  - `BreadcrumbList` для навигационной цепочки.
- [ ] **Нет Flash / Ajax-критического контента** – текст и изображения доступны в исходном HTML без JS. SSR или статическая генерация предпочтительны.
- [ ] **hreflang** – для мультиязычных страниц (ru/en) указаны теги `<link rel="alternate" hreflang="...">`.
- [ ] **Канонические теги** – `rel="canonical"` на каждой странице; дубли (одна работа в нескольких категориях) ссылаются на основной URL.

## Блок 2. Изображения (ранжирование в поиске по картинкам)

- [ ] **Атрибут `alt`** – заполнен для всех иллюстраций: описание сцены + техника + ключевые слова на языке страницы (`en` или `ru`).
- [ ] **Подписи `<figcaption>`** – название работы, техника, год, теги (жанр, стиль).
- [ ] **Имена файлов** – осмысленные, на латинице, через дефис: `dragon-portrait-2024.webp`, не `IMG_3485.jpg`.
- [ ] **Оптимизация веса** – основной формат WebP (AVIF — бонус), резервный JPEG. Превью < 500 Кб при ширине 1200–1500 px. Полноразмерные — до 2–3 Мб, открываются по требованию.
- [ ] **Ленивая загрузка** – `loading="lazy"` для всех изображений ниже сгиба; первый экран — `loading="eager"`.
- [ ] **CDN** – изображения раздаются через CDN (Cloudflare, AWS CloudFront, BunnyCDN) для снижения LCP в USA, UAE, Europe.

## Блок 3. UX/UI дизайн (современные тренды)

- [ ] **Hero-секция** – первый экран: лучшая работа или короткий ролик (loop video) на весь экран + имя + специализация + CTA-кнопка.
- [ ] **Галерея** – Masonry или CSS Grid с сохранением пропорций изображений; без обрезки горизонтальных и вертикальных работ.
- [ ] **Вайтспейс (воздух)** – отступы между работами не менее 24–32 px; шрифт основного текста от 16 px, нейтральный (Inter, Roboto, Source Sans Pro).
- [ ] **Меню** – десктоп: горизонтальное, прилипает к верху (sticky). Мобильные: бургер + выдвижная панель.
- [ ] **Тёмная тема по умолчанию** – Light mode опционально (тумблер). Тёмный фон (#0a0a0a или #121212) — стандарт для арт-портфолио.
- [ ] **Микроанимация** – при наведении: scale 1.03 + плавное затемнение с оверлеем (название, год, техника). Продолжительность 200–300 мс, ease-out.
- [ ] **Лёгкие эффекты скролла** – допускается смена прозрачности или translateY. Нет тяжёлого параллакса, который ломает CLS.
- [ ] **Прелоадер / Placeholder** – blur-up или доминирующий цвет (dominant color) как placeholder до загрузки изображения.

## Блок 4. Навигация и структура

- [ ] **Фильтр по категориям** – сортировка без перезагрузки (JS): Book Covers, Illustrations, Comics, Concept Art, Environments, Personal.
- [ ] **Страница каждого проекта (single page)** – отдельный URL с крупными иллюстрациями, описанием задачи, использованными инструментами, сроками, кнопкой «Заказать похожее».
- [ ] **Lightbox / полноразмерный просмотр** – по клику на превью открывается оверлей с HD-изображением, навигация стрелками, клавиша Escape для закрытия.
- [ ] **WIP / Process** – на странице проекта показаны этапы: скетч → blocking → детализация → финал (2–4 изображения).
- [ ] **Пагинация или «Загрузить ещё»** – бесконечный скролл не используется (или только после 2–3 страниц с кнопкой «Продолжить»).
- [ ] **Кнопка «Наверх»** – появляется после прокрутки одного экрана; плавный скролл.
- [ ] **Кастомная 404** – страница с извинением, ссылками на главные разделы и случайной работой.

## Блок 5. Конверсия и контакты

- [ ] **Короткая форма обратной связи** – Имя, Email/Telegram, Тип проекта (выпадающий список), Сообщение. Защита: honeypot-поле или простая галка.
- [ ] **Sticky-кнопка «Связаться»** – фиксированная иконка Telegram/WhatsApp в правом нижнем углу; z-index ниже меню.
- [ ] **Commissions Status** – видный блок в хедере или сайдбаре: «Слоты открыты / Закрыт по записи / Очередь на месяц».
- [ ] **Ориентир по цене** – проектные рейты (не почасовые): «Персонаж от $800», «Обложка от $500», «Концепт локации от $1200». Диапазон снижает барьер «напиши — узнаешь цену».
- [ ] **Социальное доказательство** – 2–4 отзыва с аватарами, именами, должностями и ссылками на соцсети заказчиков.
- [ ] **CTA на каждой странице проекта** – кнопка «Обсудить проект» или «Заказать похожее» в конце кейса.
- [ ] **Подписка на обновления** – форма email («Новые работы раз в месяц») для постоянных клиентов и арт-директоров.
- [ ] **Скачиваемое CV (PDF)** – кнопка в About для рекрутеров; туда спрятаны Education, Professional Development, полный Work Experience.

## Блок 6. Open Graph и социальные сети

- [ ] **Open Graph** – для каждой страницы: `og:title`, `og:description`, `og:image` (1200×630 px), `og:url`. Для работ — превью самой иллюстрации.
- [ ] **Twitter Cards** – `twitter:card="summary_large_image"`, заполнены title, description, image.
- [ ] **Мультиязычность** – интерфейс сайта на английском по умолчанию (международные клиенты Dubai, LA); русская версия — поддомен или `/ru/`.

## Блок 7. Аналитика и мониторинг

- [ ] **Google Search Console + Яндекс.Вебмастер** – подтверждение домена, отправка sitemap.
- [ ] **GA4 + Яндекс.Метрика** – цели: отправка формы, клик на Telegram, скачивание CV, просмотр 3+ работ.
- [ ] **LinkedIn Insight Tag** – приоритет для международных B2B-клиентов (ICVR, Apex Digital).
- [ ] **Meta Pixel** – для ретаргетинга в Instagram/Facebook (визуальная аудитория).
- [ ] **Пиксель ВКонтакте** – вторичный, если целевая аудитория включает РФ/Беларусь.

## Как использовать этот файл

1. Загрузите этот `.md` в диалог с Kimi.
2. Напишите: *«Ты — эксперт по SEO и дизайну сайтов для иллюстраторов. Оцени сайт [ссылка] по пунктам из загруженного файла. Для каждого пункта поставь ✅ или ❌ и добавь короткое пояснение.»*
3. Kimi прочитает файл и будет следовать чек-листу.

> **Совет:** Если у сайта несколько языковых версий, запускайте аудит отдельно для `en` и `ru` сегментов — требования к hreflang и ключевым словам отличаются.
